Addendum

Purpose: This lab demonstrates the integration of front-end and back-end technologies using ExpressJS. The goal was to create an interactive web application that receives form input, processes it server-side, and returns a dynamically generated story using user-provided data.

Process: The form in index.html collects five input fields—noun, plural noun, adjective, verb, and place—and sends them to the server using the POST method. The Express server defined in server.js receives this input via req.body and constructs a custom story string that is dynamically embedded in an HTML response. This showcases the concept of server-side rendering without using a templating engine.

Technology: The project uses Express for routing, Morgan for logging, and Node.js for execution. The front end uses semantic HTML5 and CSS for structure and design. The included “last modified” script dynamically updates the timestamp for version tracking.

Challenges: The main challenge was ensuring the correct routing between the static file directory and the POST route while keeping form submission on the same URL. Debugging was simplified using Morgan logs.

Conclusion: This project demonstrates the use of back-end JavaScript to dynamically generate and serve web content while providing a fun, interactive experience for users. The application can be easily extended to include templates or database storage in future iterations.
